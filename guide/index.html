    Sync Repos Script Documentation¶

    Overview¶

    This script, sync_repos.sh, is designed to synchronize files between source and destination directories or Git repositories. It includes features for backup, rollback, and Git operation capabilities. The script is packaged as a .deb file and can be executed using the sync_repo command from any directory.

    Features¶

    1. File Synchronization:
    2. Synchronizes files between source and destination directories.

    3. Supports Git repositories as source or destination.

    4. Backup and Rollback:

    5. Automatically creates backups before synchronization.

    6. Rollback options:

      • Rollback to the latest backup.
      • Rollback the last Git push.
      • Rollback to a specific Git commit.

    7. Git Operations:

    8. Pulls the latest changes from the source repository.

    9. Allows committing and pushing changes to the destination repository.

    10. Tracking:

    11. Maintains a JSON-based tracking file for repositories.

    12. Allows editing repository information (e.g., URL, branch, path).

    13. Logging:

    14. Logs all operations in JSON format for easy debugging and auditing.

    15. Centralized Directory:

    16. Stores all logs, backups, and tracking files in a centralized directory ($HOME/.sync_repo).

    Directory Structure¶

    The script creates and uses the following directory structure under $HOME/.sync_repo:

    ~/.sync_repo/
├── BACKUP/          # Stores backup files
├── LOGS/            # Stores log files
│   └── <date>/      # Logs are organized by date
├── SYNC_DIR/
│   ├── SOURCE_DIR/  # Source repositories
│   └── DEST_DIR/    # Destination repositories
└── repo_tracking.json  # JSON file to track repository information

    Usage¶

    Command Syntax¶

    sync_repo [OPTION]

    Options¶

    Option Description
    --rb Rollback to the latest backup.
    --rlp Rollback the last Git push.
    --rtc Rollback to a specific Git commit.
    --i Perform file synchronization.
    --edit-track Edit the repository tracking file.
    --show-ids Show operation IDs for repositories.
    --clear-backups Clear the backup folder.
    -et Alias for --edit-track.
    -id Alias for --show-ids.
    -h, --help Display usage information.

    Setup and Initialization¶

    1. Centralized Directory:

    2. The script automatically creates the centralized directory ($HOME/.sync_repo) and its subdirectories if they do not exist.

    3. Required Commands:

    4. Ensure the following commands are installed on your system:

      • tar
      • rsync
      • git
      • jq

    5. Environment Variables:

    6. The script supports a .env file for development purposes. If present, it loads environment variables from .env.

    Key Functions¶

    1. Initialization¶

    • initialize():
    • Ensures the centralized directory and required subdirectories exist.
    • Prompts the user for source and destination paths.
    • Initializes the log file and tracking file.

    2. Logging¶

    • initialize_log_file():
    • Creates a new log file in the LOGS directory.

    • Redirects script output to the log file.

    • log_message(type, message):

    • Logs messages in JSON format to the log file.

    3. Backup and Rollback¶

    • sync_files():
    • Creates a backup of the destination directory.

    • Synchronizes files from the source to the destination.

    • rollback():

    • Restores the destination directory from the latest backup.

    • rollback_last_push():

    • Rolls back the last Git push in the destination repository.

    • rollback_to_commit(commit_hash):

    • Rolls back the destination repository to a specific commit.

    4. Git Operations¶

    • clone_if_needed(input, type, var_name):

    • Clones a Git repository if it is not already tracked.

    • update_repository_branch(repo_path, new_branch, repo_url):

    • Switches or updates the branch of a Git repository.

    • git_operations():

    • Handles Git add, commit, and push operations for the destination repository.

    5. Tracking¶

    • edit_tracking_file():

    • Allows the user to edit repository information in the tracking file.

    • show_operation_ids():

    • Displays a list of tracked repositories with their operation IDs.

    6. Cleanup¶

    • clear_backups():
    • Deletes all backups in the BACKUP directory.

    Development Guidelines¶

    1. Adding New Features:
    2. Follow the existing structure for logging and error handling.

    3. Ensure new features integrate with the centralized directory structure.

    4. Testing:

    5. Test the script in both development and production environments.

    6. Use mock repositories for testing Git operations.

    7. Error Handling:

    8. Use log_message to log errors and warnings.

    9. Exit the script gracefully on critical errors.

    10. Documentation:

    11. Update this README.md file whenever new features are added.

    Troubleshooting¶

    1. Permission Issues:

    2. Ensure the script has write access to $HOME/.sync_repo.

    3. Missing Commands:

    4. Install missing commands using your package manager (e.g., sudo apt install jq).

    5. Git Errors:

    6. Verify that the source and destination repositories are accessible and have the correct permissions.

    7. Tracking File Corruption:

    8. If the repo_tracking.json file is corrupted, delete it and let the script recreate it.

    Future Enhancements¶

    1. Multi-User Support:

    2. Add support for system-wide usage with /var/lib/sync_repo.

    3. Improved Logging:

    4. Add log rotation and archiving for older logs.

    5. Web Interface:

    6. Develop a web-based interface for managing repositories and tracking files.

    7. Advanced Rollback:

    8. Add options for partial rollbacks (e.g., specific files or directories).
