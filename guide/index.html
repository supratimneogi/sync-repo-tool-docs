    Sync Repos Script Documentation¶

    üìã Overview¶

    The syncRepoV3_LTS.sh script is a powerful tool designed to synchronize files between source and destination directories or Git repositories. It provides robust features for backup, rollback, and Git operations. After installation, you can run it from any directory using the sync_repo command.

    ‚ú® Features¶

    üìÇ File Synchronization¶

    • Seamlessly sync files between directories or Git repositories
    • Support for both local paths and remote Git repositories
    • Intelligent handling of Git branches

    üîÑ Backup and Rollback¶

    • Automatic Backups: Creates backups before any synchronization
    • Multiple Rollback Options:
    • --rb: Rollback to latest backup
    • --rlp: Rollback the last Git push
    • --rtc: Rollback to a specific Git commit

    üîß Git Integration¶

    • Pull latest changes from source repositories
    • Commit and push changes to destination repositories
    • Branch management and switching

    üìä Repository Tracking¶

    • Track repositories with unique operation IDs
    • Edit repository information with user-friendly interface
    • View all tracked repositories in a formatted display

    üìù Comprehensive Logging¶

    • Detailed JSON-based logs for all operations
    • Logs organized by date for easy reference
    • View logs directly from the command line

    üóÇÔ∏è Directory Structure¶

    üöÄ Quick Start Guide¶

    Installation¶

    # Install required dependencies
sudo apt install git rsync jq tar

# Run the script directly
bash curl -L -u "<your-username>:<your-app-password>" -O "https://api.bitbucket.org/2.0/repositories/supratimneogi007/syncmaster/downloads/sync-repos_2.7.3_all.deb" && sudo apt install ./sync-repos_2.7.3_all.deb

    Basic Usage¶

    # Perform file synchronization
sync_repo --i

# Rollback to latest backup
sync_repo --rb

# Edit repository tracking information
sync_repo --edit-track

    üìö Command Reference¶

    Command Alias Description Example
    --i Perform file synchronization sync_repo --i
    --rb Rollback to latest backup sync_repo --rb
    --rlp Rollback last Git push sync_repo --rlp
    --rtc Rollback to specific commit sync_repo --rtc
    --edit-track -et Edit repository tracking sync_repo --edit-track
    --show-ids -id Show operation IDs sync_repo --show-ids
    --clear-backups Clear backup folder sync_repo --clear-backups
    --show-logs Display available logs sync_repo --show-logs
    --version Show script version sync_repo --version
    --help -h Display help information sync_repo --help

    üìù Practical Examples¶

    Example 1: Syncing Between Two Git Repositories¶

    # Start synchronization process
sync_repo --i

# When prompted, enter Git URLs:
# Source: git@github.com:username/source-repo.git
# Destination: git@github.com:username/destination-repo.git
# Follow the prompts to complete the sync

    Example 2: Using Operation IDs for Quick Access¶

    # First, view available operation IDs
sync_repo --show-ids

# Then use the operation ID for syncing
sync_repo --i
# When prompted, select option 2 for operation ID
# Enter the ID shown from --show-ids command

    Example 3: Rollback After a Failed Sync¶

    # If a sync operation causes issues, roll back
sync_repo --rb

# Confirm the rollback when prompted

    üîç Advanced Usage¶

    Working with Branches¶

    The script allows you to specify and switch branches:

    1. Edit a repository's branch: bash sync_repo --edit-track # Select the repository and choose option 3 to edit branch

    2. When cloning a new repository, you'll be prompted for branch name: Enter branch name for source repository [main]: develop

    Environment Variables¶

    The script supports a .env file for development purposes:

    # Sample .env file

BASE_DIR=/custom/path/to/sync_repo

    Required Dependencies¶

    Ensure the following commands are installed: - tar: For creating backups - rsync: For file synchronization - git: For repository operations - jq: For JSON parsing

    üîß Troubleshooting¶

    Common Issues and Solutions¶

    Issue Possible Cause Solution
    Permission denied Lack of write access Ensure you have write permissions to ~/.sync_repo
    Git operation failed Repository access issues Check repository permissions and credentials
    Command not found Missing dependencies Install required dependencies (tar, rsync, git, jq)
    Tracking file errors Corrupted JSON Run sync_repo --edit-track to fix or recreate tracking file
    Branch switching failed Non-existent branch Ensure the branch exists or create a new one when prompted

    Log Analysis¶

    View detailed logs to diagnose issues:

    sync_repo --show-logs
# Select date and specific log file when prompted
