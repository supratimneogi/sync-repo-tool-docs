    Usage¶

    This guide provides detailed instructions on how to use the sync-repos tool effectively after installation.

    Running the Tool¶

    1. Open a terminal and execute the following command: bash sync-repos
    2. Alternatively, use the interactive mode: bash sync-repos --i
    3. Follow the prompts to specify the source and destination paths: bash Enter source path (default: /home/user/.sync-repos/source): /path/to/source Enter destination path (default: /home/user/.sync-repos/dest): /path/to/dest

    Core Functionality¶

    The sync-repos tool supports synchronization across the following:

    • Local Directories: Copies files and directories while preserving structure, permissions, and timestamps.
    • Git Repositories: Clones or pulls from Git URLs, with rollback and commit-based synchronization options.
    • OP_IDs: Custom identifiers for predefined synchronization tasks (if configured).

    Options¶

    • --rb: Roll back to a previous state.
    • --rlp: Roll back the last push (Git only).
    • --rtc: Roll back to a specific commit (Git only).
    • --edit-track: Edit repos track files.
    • --show-ids: Views repos track files.
    • --help or -h: Display the help menu.

    Synchronization Process¶

    Input Handling¶

    Accepts source and destination inputs, which can be directories, Git URLs, or OP_IDs.

    Validation¶

    Verifies that the provided paths or URLs are valid and accessible.

    Synchronization¶

    • For Directories: Uses rsync to efficiently copy files.
    • For Git Repositories: Clones or pulls changes, supporting rollback to prior states.

    Logging and Backup¶

    Generates logs and backups for auditing and recovery.

    Command-Line Options¶

    Customize the tool’s behavior using the following options:

    Option Description
    --rb Rolls back to a previous synchronization state (directory or Git).
    --rlp Rolls back the last push (Git repositories only).
    --rtc Rolls back to a specific commit (Git repositories only).
    --edit-track Edit repos track files..
    --show-ids Views repos track files.
    --help, -h Displays the help menu with usage details.

    Usage Examples¶

    Rollback to a Previous State¶

    sync-repos --rb

    Rollback Last Git Push¶

    sync-repos --rlp

    Rollback to a Specific Commit¶

    sync-repos --rtc <commit-hash>

    Runtime Artifacts¶

    The tool creates and manages files in ~/.sync-repos/ to track synchronization states, logs, and backups. Understanding these artifacts aids in troubleshooting and recovery.

    Directory Structure¶

    ~/.DevOps_tool/
├── LOGS/
│   └── <date>/            # Logs for each synchronization run
├── BACKUP/                # Backups of synchronized data
├── SYNC_DIR/              # Cloned Git repositories
    └── repo_tracking.json # Tracks synchronization history

    Logs¶

    • Location: ~/.sync-repos/logs/<date>/
    • Purpose: Records actions, errors, and warnings from each synchronization.

    Backups¶

    • Location: ~/.sync-repos/backups/
    • Purpose: Stores pre-synchronization backups for rollback.

    Cloned Repositories¶

    • Location: ~/.sync-repos/SYNC_DIR/
    • Purpose: Holds cloned Git repositories for state management.

    Tracking File¶

    • File: repo_tracking.json
    • Purpose: Logs synchronization history, including timestamps, paths, and Git commit hashes.

    Advanced Usage¶

    Synchronizing Git Repositories¶

    The tool offers enhanced Git support:

    • Cloning: Clones a repository if the destination doesn’t exist.
    • Pulling: Updates an existing destination with the latest changes.
    • Rollback: Reverts to previous commits or states using --rb, --rlp, or --rtc.

    Example: Sync a Git Repository¶

    sync-repos
Enter source path: https://bitbucket.org/supratimneogi007/syncmaster.git
Enter destination path: /path/to/local/repo

    Example: Roll Back to a Commit¶

    sync-repos --rtc abc123def456

    Using OP_IDs¶

    For predefined tasks identified by OP_IDs, input the ID instead of paths:

    sync-repos --i
Enter source path: OP_ID:12345

    The tool maps the OP_ID to configured source and destination paths.

    Best Practices¶

    • Backup Regularly: While the tool creates backups, manually back up critical data beforehand.
    • Review Logs: Inspect ~/.sync-repos/logs/ after runs to verify success.
    • Stay Updated: Use the latest sync-repos version for improvements and fixes.
